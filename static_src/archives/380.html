C++17 Standards (上)

Posted by By zdl 2026年5月9日No Comments

 

在C++的迭代历史中C++17也是一个重大的更新版本，他引入了新的语言和特性库。也是在2017年发布的。

1. 结构化绑定

🌐网站资料：

结构化绑定和范围for循环一样都是是一种语法糖，它允许你从一个数组种、结构体、元组或类似的拥有多个成员数据的对象中，一次性申明并初始化多个数据；特别是在处理std::pair 和 std::tuple这样的数据时。

  • 🧐结构化绑定的数量一定要和表达式中的元素的数量匹配
  • 结构化绑定的变量的生命周期与绑定的对象的相同。
  • 结构化绑定的变量时时表达式种对应的元素的拷贝或副本
  • 结构化绑定不能用于类的私有成员，除非在类的内部使用结构化绑定
  • 结构化绑定不支持嵌套

下面展示一段代码，我们在代码中搞清楚在实践中的一些问题：


#include<iostream>
#include<map>
#include<string>
using namespace std;
int main()
{
	// 1. 解包数组 
	int arr[2] = { 1, 2 };
	auto [x, y] = arr; // x = 1, y = 2			
	cout << x << endl;
	cout << y << endl;
	// 需要注意的是结构化绑定是将arr中的值拷⻉给x和y 
	// 也就是说x和y的改变，不影响数组中的值 
	x++;
	cout << arr[0] << endl;
	// 这⾥加引⽤，就可以做到xx和yy是数组中对应值的别名 
	auto& [xx, yy] = arr;
	xx++;
	cout << arr[0] << endl;
	// 2. 解包元组(std::tuple) 
	std::tuple<int, double, std::string> t(1, 2.3, "hello");
	auto [a1, b1, c1] = t;
	auto& [a2, b2, c2] = t;
	auto&& [a3, b3, c3] = move(t);
	//auto& [a4, b4] = std::make_tuple(1, 2); // 编译报错 
	auto&& [a5, b5] = std::make_tuple(1, 2);
	 // 3.解包结构体/类 
		struct Point {
		void func()
		{
			auto [x1, y1] = *this; // x1=1.1, y1=2.							
			cout << x1 << endl;
			cout << y1 << endl;
		}
		//private:
		double x;
		double y;
	};
	// 如果成员变量是私有的，则⽆法做结构化绑定 
	Point p{ 1.1, 2.2 };
	auto [x1, y1] = p; // x1=1.1, y1=2.			
	cout << x1 << endl;
	cout << y1 << endl;


	// 结构化绑定不⽀持嵌套绑定 
	//auto [[mm1, mm2], nn1] = std::make_tuple(p, 1); // 报错 
	auto [m1, n1] = std::make_tuple(p, 1);
	// 实践使⽤场景：范围for叠加结构化绑定使⽤遍历map 
	std::map<std::string, int> m = { {"xxxx", 1}, {"yyyy", 2} };
	for (const auto& [key, value] : m)
	{
		std::cout << key << ": " << value << "n";
	}
	return 0;
}

 

2. inline 变量

C++17新增的inline 变量这是一个非常重要的特性，它极大地简化了头文件中全局变量和静态成员变量的管理和使用。

为什么要引入inline变量？ — 解决 ODR的使用问题

  • 在C++17之前，想要在头文件中定义一个变量（比如一个全局变量），就会遇到经典的难题：ORD（one Define Rule）单定义规则
  • .h定义全局变量的时候，多个.cpp包含就会被定义很多次，会链接报错。const和static全局变量默认有内部链接（InternalLinkage), 不会报错但是这意味这每一个包含.h的.cpp文件（翻译单元）， 都会自己常见一个独立的bufferSize and cacheSize变量。虽然这些变量的值和名称都一致但是他们的地址事不一样的。这违背了“全局唯一变量”的初衷。
  • 如果你想那个要一个真正的、所有翻译单元共享的全局变量，你必须在头文件中extern申明，再在某一个.cpp文件中单独定义，这非常的麻烦。
// Cache.h
#include<iostream>
using namespace std;
// 在多个翻译单元(.cpp⽂件)中包含此头⽂件会导致链接错误 
// int bufferSize = 1024; // 链接报错 
const int bufferSize = 1024; // 每个.cpp⽂件都会有⾃⼰的`bufferSize`副本，这不是我们想要的
const int cacheSize = 4 * 1024;
void func();
// Cache.cpp
#include "Cache.h"
void func()
{
	cout << &bufferSize << endl;
	cout << &cacheSize << endl;
}
// Testc.pp
#include"Cache.h"
int main()
{
	//cout << bufferSize << endl;
	cout << &bufferSize << endl;
	cout << &cacheSize << endl << endl;
	func();
	return 0;
}

 

为了解决这样的问题，inline诞生了：

  • inline关键字用于变量时，主要含义是：允许（甚至要求）在多个翻译单元中定义相同的变量，并且保证这些变量指向同一个实体（地址相同）。
  • 编译器会从众多的定义中挑一个，并丢弃其他的，从而确保整个程序中只有一个该变量的实例。这完美契合了在头文件中定义变量的要求！

类的静态成员变量的定义和初始化

  • C++17之前，类的静态成员变量，要做声明和定义的分离，这非常的麻烦。
// Cache.h
class PhoneCall{
	public:
	static std::string name; // 声明
	static const int version = 1; // 仅对<strong>整形static const 允许在类内定义和初始化</strong>
};
// Cache.cpp
std::string PhoneCall::name = "张三"; // 定义初始化

 

  • C++17之后，使用inline，你可以在声明的同时直接初始化，所有的定义相关的工作都在头文件中完成，inline变量最常用、最直观的应用场景，他彻底简化了乐成员变量的初始化方式。
// Cahe.h
class PhoneCall{
	public:
	inline static std::string name = "张三"; // 定义初始化
	static const int version = 1; // static const int 类型还是可以在类中直接声明和定义！
};

 

3. if/switch初始化语句

C++17允许在if和switch语句中声明并初始化变量，这个语法将辅助分支控制的变量的生命周期控制到了分支控制体的内部，保证了代码的简洁性 😋


#include <iostream>
#include <map>
#include <string>
// ⽰例1: if语句中的初始化 
void ifWithInit() {
	std::map<std::string, int> m = { {"Alice", 25}, {"Bob", 30} };
	// 在if条件中初始化变量 
	if (auto it = m.find("Alice"); it != m.end()) {
		std::cout << "Found Alice, age: " << it->second << std::endl;
	}
	else {
		std::cout << "Alice not found" << std::endl;
	}
	// 变量it的作⽤域仅限于if-else块 
}
int checkValue(int v) {
	return v % 2;
}
// ⽰例2: switch语句中的初始化 
void switchWithInit(int value) {
	// 在switch条件中初始化变量 
	switch (auto status = checkValue(value); status) {
	case 0:
		std::cout << "Status is zero" << std::endl;
		break;
	case 1:
		std::cout << "Status is one" << std::endl;
		break;
	default:
		std::cout << "Unknown status: " << status << std::endl;
	}
	// status的作⽤域仅限于switch块 
}
int main() {
	ifWithInit();
	switchWithInit(42);
	return 0;
}

 

4. 强制省略拷贝（返回值优化标准)

技术）

🌐网站资料：

拷贝省略（Copy Elision）是编译器的一种优化技术，目的是为了避免创建不需要的临时对象，从而提升性能。常见的拷贝拷贝省略形式就是返回值优化（unnamed return value optimization, URVO）和命名返回值优化（Namee Return Value Optimization, NRVO）.

其实这个在我们的前面C++标准学习中已经介绍过了，这里我们就简单的回顾一下，本质就是优化省略临时对象以前C++标准并未规定，是编译器选择实现，C++17开始强制规定必须实现URVO,但是NRVO不是强制的，还是需要看编译器的实现，不过现在主流的编译器几乎都会进行NRVO优化，具体细节可以看一下C++11的的右值引用和移动构造，在哪里我们进行了解释。

#include <iostream>
struct Noisy
{
	Noisy() { std::cout << "constructed at " << this << 'n'; }
	Noisy(const Noisy&) { std::cout << "copy-constructedn"; }
	Noisy(Noisy&&) { std::cout << "move-constructedn"; }
	~Noisy() { std::cout << "destructed at " << this << 'n'; }
};

Noisy f()
{
	Noisy v = Noisy(); // (until C++17) copy elision initializing v from a temporary;
	// the move constructor may be called
			// (since C++17) "guaranteed copy elision"
	return v; // copy elision ("NRVO") from v to the result object;
	// the move constructor may be called
}

void g(Noisy arg)
{
	std::cout << "&arg = " << &arg << 'n';
}

int main()
{
	Noisy v = f(); // (until C++17) copy elision initializing v from the result of f()
	// (since C++17) "guaranteed copy elision"
	std::cout << "&v = " << &v << 'n';
	g(f()); // (until C++17) copy elision initializing arg from the result of 
	f();
	// (since C++17) "guaranteed copy elision"
}

 

在这里可以看出省略临时拷贝的作用，在实际工程中，这样的改进可以极大的减少拷贝，提高运行时速率。

  • 在C++14的标准逻辑下，即使编译器进行优化，但是代码必须在语法上满足所有拷贝、移动构造函数可访问的要求。因此，上面的代码在在C++14之前没有办法达到效果。

在linux的编译环境下，在编译指令下加入-std=c++14就可以，这样就可以证明，在C++17省略拷贝是C++17强制开启的。

  • 在C++17中，下面段代码是合法的。因为根据新的标准， return NonMoveable() , 和 NonMoveable nm = make() , 这两条语句根本不会调用任何拷贝或者是移动构造函数。既然不用调用，自然就不需要他们可访问。编译器直接会在目标的内存位置构造对象！
struct NonMoveable{
	NonMoveable() = default;
	NonMoveable(const NonMoveable&) = delete; // 禁止拷贝
	NonMoveable(NonMoveable&&) = delete; // 禁止移动
} ;

NonMoveable make(){

	return NonMoveable(); // 在C++14中会直接编译报错
	// 因为在语言逻辑上要求调用已删除的移动构造函数！
}

int main(){
	NonMoveable nm = make(); // 同样也会错！！
}

 

5. if constexpr

if constexpr是C++17引入的一条条件编译的语句，他允许在编译时根据常量表达式的结果决定编译那部分代码，未选择的分支代码不会编译成指令，直接丢弃。

if 与 if constexpr的区别

基本语法：

if constexpr(常量表达式){
	// 条件为 true时编译的代码
}else if constexpr(常量表达式){

}else {
	
}

 

下面是一段示例代码： 

#include <iostream>
#include <type_traits>
// ⽰例1: 基本⽤法 
template < typename T>
auto printTypeInfo(const T& value) {
	if constexpr (std::is_integral_v<T>) {
		std::cout << value << " is an integral type" << std::endl;
	}
	else if constexpr (std::is_floating_point_v<T>) {
		std::cout << value << " is a floating point type" << std::endl;
	}
	else if constexpr (std::is_pointer_v<T>) {
		std::cout << "Pointer to " << *value << std::endl;
	}
	else {
		std::cout << value << " is some other type" << std::endl;
	}
}
// ⽰例2: 在模板元编程中使⽤ 
template < typename T, typename... Args>
void printArgs(T&& arg, Args&&... args) {
	std::cout << arg;
	if constexpr (sizeof...(args) > 0) { // 折叠表达式，下面会详细讲解
		std::cout << ", ";
		printArgs(std::forward<Args>(args)...);
	}
	else {
		std::cout << std::endl;
	}
}
// ⽰例3: 类型分发 
template < typename T>
void process(T value) {
	if constexpr (std::is_same_v<T, int>) {
		std::cout << "Processing int: " << value * 2 << std::endl;
	}
	else if constexpr (std::is_same_v<T, std::string>) {
		std::cout << "Processing string: " << value.length() << " chars" <<
			std::endl;
	}
	else {
		static_assert(std::is_same_v<T, int> || std::is_same_v<T, std::string>,
			"Unsupported type");
	}
}
int main() {
	printTypeInfo(42);
	printTypeInfo(3.14);
	int x = 10;
	printTypeInfo(&x);
	printTypeInfo("hello");
	printArgs(1, 2.5, "three", '4');
	process(100);
	process(std::string("test"));
	//process(3.14); // 编译错误，因为static_assert 
	return 0;
}

 

6. 折叠表达式

  • C++17之前，当我们编写可看参数模板（Variadic Template）时，通常需要使用递过来实现参数包的展开。
#include <iostream>
using namespace std;
// C++11/14的方式：递归模板
template<class T>
T sum(T t){
	return t;
}

template <class T, class... Args>
T sum(T first , Args... args){
	return first + sum(args ...); // 递归调用
}

void ShowList(){

	// 编译器时递归的终止条件， 参数包是0个时，直接匹配这个函数
	cout << endl;
}

template <class T, class ...Args>
void ShowList(T x, Args... args)
{
// 这里表示的是， args是N个参数的参数包
// 调用 ShowList， 参数包的第一个值传递给了这个函数（x），
// 剩下N - 1个参数分配给了第二个参数包
	cout << x << " ";
	ShowList(args ...);
}

template <class ...Args>
void Print(Args... args)
{
	ShowList(args...);
}
int main(){
	auto total = sum (1, 2, 3, 4, 5);
	cout << "total : " << total << endl;
	Print(1, string("hello world"), 2.2);
}

 

  • C++17中的折叠表达式就是为了优雅的解决这个问题而生的。他允许你使用简洁的缘分直接对参数包中的参数进行二次操作符的“折叠计算”， 有了折叠表达式上面的代码可以优化为如下：
#include <iostream>
using namespace std;

template <class T, class.. Args>
void Print
int main(){
	cout << sum(1, 2, 3, 4, 5) << endl;
	Print(1, string("xxxxx"), 2.3);
	return 0;
}

 

折叠表达式语法的参考资料：

折叠表达式是C++17引入的一个强大特性，他简化了对参数包的操作，使得处理可变参数模板时代码更加的简介高效。他允许将一个二元操作应用在所有的参数包参数中，将其“折叠”成一个单一的结果。他主要运用在模板元编程中，特别是可变参数模版的场景。

他的语法形式主要有四种（以二元操作符op为例）
  • 一元右折叠（pack op ...）
    • 展开形式
    • 计算顺序：从右往左
  • 一元左折叠 （... op pack）
    • 展开形式 :
    • 计算顺序：从左往右
  • 带初始值的二元右折叠 (pack op ... op init)
    • 展开形式：
    • 计算顺序：从右往左
  • 带初始值的二元左折叠 (init op ... op pack)
    • 展开形式：
    • 计算顺序： 从左往右
#include <iostream>

using namespace std;

// 1. 一元左折叠
template<class ...Args>
bool all(Args... args){
	return (... && args);
	// return ()
}
// 2. 一元有折叠
template <class ...args>
int sum (Args... args){
	return (args + ...);
}

int main(){
	cout << all(1, 0, 1, 1, 1, 1) << endl; // 输出的结果是0
	cout << sum(1,2,3,4,5,6) << endl; //  21
}

 

#include<iostream>
#include<vector>
#include<string>
// 3、⼆元左折叠 
template < typename... Strings>
std::string concat_left(Strings... strs) {
	return (std::string("%%") + ... + strs);
	// 展开形式：((("%%" + str1) + str2)+ str3) 
}
// 4、⼆元右折叠，初始值为空字符串 
template < typename... Strings>
std::string concat_right(Strings... strs) {
	return (strs + ... + std::string("%%"));
	// 展开形式：str1 + (str2 + (str3 + "%%")) 
}
// ⼆元左折叠 
template < typename... Args>
void print(Args&&... args) {
	(std::cout << ... << args) << 'n';
	// 等价于 (((std::cout << arg1) << arg2) << ...) << argN; 
}
// ⼀个更⾼级的技巧：使⽤逗号运算符和lambda来添加分隔符 
template < typename... Args>
void print_with_separator(Args&&... args) {
	auto print_elem = [](const auto& x) {
		std::cout << x << " ";
		};

	(..., print_elem(args)); // ⼀元左折叠： 
	// ((print_elem(arg1),print_elem(arg2)), print_elem(arg3))
	std::cout << std::endl;
}
template < typename T, typename... Args>
void push_back_vec(std::vector<T>& v, Args&&... args)
{
	(v.push_back(std::forward<Args>(args)), ...);
}
int main() {
	std::cout << concat_left("x", "y", "z") << "n";
	std::cout << concat_right("x", "y", "z") << "n";
	print(1, "hello", 3.14); // 输出 "1hello3.14" 
	print_with_separator(1, "hello", 3.14); // 输出 "1 hello 3.14" 

	std::vector<int> v;
	push_back_vec(v, 1, 2, 3, 4);
	for (int i : v)
		std::cout << i << ' ';
	std::cout << 'n';
	return 0;
}

当参数包为空的情况

当参数包位空时，折叠表达式的行为取决于操作符和是否有初始值：

对于一元折叠（没有初始值）：
  • 如果参数包使用的是&&， 他的值是true .
  • 如果对空包使用||， 他的值是false
  • 如果对空包使用的是, , 他的值是 void() 。
对于二元折叠表达式（有初始值）
  • 如果参数包是空，表达式的结果就是初始值init， 这使得带初始值的版本更加的安全和通用 
    
#include <iostream>
using namespace std;
template<class ...Args>
bool all(Args... args) {
	return (... && args);
	// 返回true
}

template <class ...Args>
int sum(Args... args) {
	//return (... + args); // 传递空参数包就直接报错。
	 return (0 + ... + args); // 带有初始值参数的版本就会更加的安全
}

int main() {

	cout << all() << endl;
	cout << sum() << endl;
}

     

7. 类模版参数自动推导

🌈 🌈 😋

  • C++17 引入了类模板参数的推导（Class Template Argument Deduction, CTAD）功能，他允许编译器根据构造参数自动推导模板参数的类型，从而简化类模板的实例化

#include <iostream>
#include <vector>
#include <tuple>
#include <array>
#include <string>
// ⽰例1: 标准库容器的⾃动推导 
void containerDeduction() {
	// C++17之前需要指定模板参数 
	std::vector<int> v1 = { 1, 2, 3 };
	// C++17可以⾃动推导 
	std::vector v2 = { 4, 5, 6 }; // 推导为vector<int> 
	std::vector v3{ "hello", "world"}; // 推导为vector<const char*> 
	std::vector v4(10, std::string());
	std::vector v5(v1.begin(), v1.end());
	// std::vector v6; // 报错 
	std::cout << typeid(v2).name() << std::endl;
	std::cout << typeid(v3).name() << std::endl;
	std::cout << typeid(v4).name() << std::endl;
	std::cout << typeid(v5).name() << std::endl;
	// std::cout << typeid(v6).name() << std::endl;
						// 类似地适⽤于其他容器 
	std::array a = { 1, 2, 3 }; // 推导为array<int, 3> 
	std::pair p(1, 2.0); // 推导为pair<int, double> 
	std::tuple t(1, 2.0, "three"); // 推导为tuple<int, double, const char*> 
	std::cout << "v2 size: " << v2.size() << std::endl;
	std::cout << "a size: " << a.size() << std::endl;
	std::cout << "p first: " << p.first << ", second: " << p.second <<
		std::endl;
}
int main() {
	containerDeduction();
	return 0;
}

 

8. 非类型模板参数

  • 非类型模板参数是一个具有特定类型的常量值，在C++17 之前，非类型模板参数的类型必须被明确指定，C++17 之后我们使用auto 关键字就可以作为非类型模板的占位符，编译器会根据实例化时提供的实参自动推导出该参数的类型。
  • 在C++20 之前，非类型模板参数的类型被严格的限制在：整形、枚举类型、指针类型、左值引用、<strong>std::nullptr_t</strong> , <strong>C++20</strong> 打破了这一限制，允许使用浮点数类型，允许你使用满足特定条件的类类型（称为 字面量类类型）的对象作为编译器常量，并传递给模板。字面量类类型C++20 会具体讲解，简单来说他就是一个可以在编译期完全构造和析构的简单的、透明的数据结构，如std::array ，简单的自定义结构体 struct Point{int x, y;}; 等

#include <iostream>
#include <string>
#include <string_view>
#include <array>
#include <typeinfo>
template < auto Value>
void printValue() {
	//std::cout << "Value: " << Value << std::endl;
	std::cout << "Type: " << typeid(Value).name() << std::endl; // 打印类型名（编译器相关）
		std::cout << "---" << std::endl;
}
// ⼀个简单的字⾯量类类型 
struct Point {
	int x;
	int y;
	// 编译器会为我们⽣成⼀个隐式的 constexpr 构造函数 
		// 析构函数也是隐式 constexpr 的 
};
template < auto... Values>
struct ValueList
{
	ValueList()
	{
		// 逗号折叠表达式 
		((std::cout << Values << ' '), ...);
	}
}; // ⼀个编译期的值列表 
const char arr[] = "hello";
int main() {
	printValue<42>(); // auto 被推导为 int 
	printValue<3.14>(); // auto 被推导为 double C++20开始才⽀持 
	printValue<'A'>(); // auto 被推导为 char 
	printValue<true>(); // auto 被推导为 bool 
	printValue<nullptr>(); // auto 被推导为 nullptr_t 
	// printValue<"hello">(); // 错误：字符串字⾯量不能作为⾮类型模板参数 
	printValue<arr>(); // auto 被推导为 const char* 
	// C++20开始⽀持字⾯量类类型作为⾮类型模板参数 
	printValue < Point{ 10, 20 } > ();
	constexpr std::array arr{ 1, 2, 3, 4, 5 };
	printValue<arr>();
	// 计算值列表中的值的类型 
	ValueList<1, 2.2, 'a'> vl;
	//printValue<string("Hello")>(); // 报错 
}

 

9. 嵌套命名空间定义

  • C++17 引入了简化嵌套空间定义的语法，使得定义多层的命名空间更加的简洁方便。这是对传统的命名空间定义方式的一种语法糖改进，本身使用起来很贱但，这里就直接展示一下他的使用样例：

#include <iostream>
// C++17之前的嵌套命名空间 
namespace A {
	namespace B {
		namespace C {
			void foo() {
				std::cout << "Old nested namespace style" << std::endl;
			}
		}
	}
}
// C++17的简洁嵌套命名空间 
namespace A::B::C {
	void bar() {
		std::cout << "New nested namespace style" << std::endl;
	}
}
namespace A::B {
	void qux() {
		std::cout << "Partially nested namespace" << std::endl;
	}
}
int main() {
	A::B::C::foo();
	A::B::C::bar();
	A::B::qux();
	return 0;
}

 

10. __has_include

  • __has_include是C++17 引入的预处理功能，用于在编译时期检查某一个头文件是否可以被包含
  • __has_include(<头文件>)或__has_include("头文件"), 他发峰会的是一个可以被预处理器解释的整形常量表达式，1表示头文件存在且可以被包含0表示头文件不可用
  • 他主要的目的是为了编写跨平台或者是跨编译器的可移植性代码。不同的系统或者是编译器版本可能提供不同的头文件库。使用__has_include，你可以有条件的包含头文件或者是采取备选方案，从而避免编译错误。

这个是一个非常实用的预处理工具，下面我们直接看看他的示例代码：


#include <iostream>
// ⽰例1:检查标准库头⽂件

#if __has_include(<optional>)
#include <optional>
#define HAS_OPTIONAL 1
#else
#define HAS_OPTIONAL 0
#endif
//⽰例2:检查⾃定义头⽂件

#if __has_include("my_header.h")
#include "my_header.h"
#define HAS_MY_HEADER 1
#else
#define HAS_MY_HEADER 0
#endif
// ⽰例：检查是否⽀持⽂件系统相关库

#if __has_include(<filesystem>)
#include <filesystem>
namespace fs = std::filesystem;
#elif __has_include(<experimental/filesystem>)
#include <experimental/filesystem>
namespace fs = std::experimental::filesystem;
#else
#error "需要filesystem⽀持"
#endif
int main() {
	std::cout << "Optional support: " << HAS_OPTIONAL << std::endl;
	std::cout << "My header support: " << HAS_MY_HEADER << std::endl;
	//实际使⽤

#if HAS_OPTIONAL
		std::optional<int> opt = 42;
	std::cout << "Optional value: " << *opt << std::endl;
#else
		std::cout << "Optional not available" << std::endl;
#endif
	return 0;
}

 

11. 属性

  • 属性是代实体（变量、函数、类等）额外添加信息的一种机制，这些信息可以帮助编译器进行优化、生成警告和错误，以及知道静态分析工具。
  • 在C++11之前，各个编译器都是用自己的缘分。
// GCC/Clang:
__attribute__((noreturn)) void func() {}
__attribute__((deprecated)) void old_func() {}

// MSVC
__declspec(noreturn) void func() {}
__declspec(deprecated) void old_func() {}
// C++11以后，使⽤统⼀的标准库属性

[[noreturn]] void func() {}
[[deprecated]] void old_func() {} // 注意：deprecated是C++14加⼊的

 

  • C++11将属性语法标准化引入了 [[ ]] 语法，但是只提供了两个属性。在后续的标准中，属性逐步丰富：参考网页资料
[[attribute]] // 单个属性
[[attribute1. attribute2]] // 多个属性
[[namesapce::attribute]] // 带有命名空间的属性

 

🌈<strong>[[noreturn]]</strong> (<strong>C++11</strong>)

指示函数不会返回到他的调用点，主要用于优化和错误检测。通过明确的告知表一起不会反悔，编译器可以优化代码的逻辑，例如跳过某一些无效的代码执行等。标记为noreturn的函数返回时，编译器会提示警告。

🕯️[[carries_dependency]](C++11)

该记得并发属性，他是为专家级并发提供的工具，但是对于 的开发者来说，使用std::atomic的默认内存序或者是其他的高并发工具是更加明智的选择

#include <cstdlib>
#include <iostream>
#include <stdexcept>
// 1. 终⽌程序的函数

[[noreturn]] void fatal_error(const std::string & message) {
	std::cerr << "Fatal error: " << message << std::endl;
	std::exit(EXIT_FAILURE);
	//不需要return语句

}
// 2.总是抛出异常的函数

[[noreturn]] void throw_runtime_error(const char* message) {
	throw std::runtime_error(message);
	// 不会执⾏到这⾥

}
// 3. ⽆限循环的函数

[[noreturn]] void event_loop() {
	while (true) {
		// 处理事件...
	}
	// 不会执⾏到这⾥

}
// 编译器优化⽰例

int main() {
	fatal_error("error exit");
}

 

❤️‍🔥[[deprecated]](C++14)

  • 它适用于标记某一个实体（如函数、类、变量、类型等）已被弃用，不建议再使用。当编译器遇到使用被标记为deprecated的实体的时候，就会发出编译警告！
#include <iostream>
// 标记⼀个函数为弃⽤

[[deprecated]]
void old_function() {

}

// 可以提供⼀条⾃定义的警告信息
[[deprecated("Use the new_function() instead, which is safer and faster.")]]
void legacy_function() {

}
// 也可以标记类、类型别名、变量等

[[deprecated]]
typedef char* CString;
struct [[deprecated]] OldStruct {};
[[deprecated]]
int obsolete_variable;
[[deprecated("This function is prone to errors. Use calculate() instead.")]]
int compute() {
	return 42;
}
// deprecated也可以⽤于类型、变量、枚举等

// v1.0
[[deprecated("这是v1.0版本，请使用v2.0改⽤connect(url, options)")]]

{
}
// v2.0
void connect(std::string url, int options)
{
}
int main() {
	
	int x = compute(); // 编译时会产⽣警告或错误

	std::cout << x << std::endl;
	CString s = nullptr;

这个属性也往往用的更多，在版本更新迭代中经常使用到。

  • C++17对属性进行了重大的拓展。引入了三个非常重要的属性，并极大的扩展了属性的适用范围。

💎[[fallthrough]](C++17)

用于在switch语句中显示的表明从一个case标签“贯穿”到下一个case标签是有意为之，以避免编译器发出警告

 

🍬[[nodiscard]](C++17)

用于标记一个函数的返回值非常的重要，调用者不应该忽略他。如果调用者没有使用返回值，编译器会发出警告。C++20允许添加原因信息[[nodiscard("xxx")]]

 

🌻 [[maybe_unused]](C++17)

用于一直编译器从未使用实体发出的“未使用变量/参数”警告。

#include <iostream>
// [[fallthrough]] 属性

void checkValue(int x) {
	switch (x) {
	case 1:
		std::cout << "Case 1" << std::endl;
		[[fallthrough]];  // 故意不break，明确告知编译器

	case 2:
		std::cout << "Case 2" << std::endl;
		break;
	default:
		std::cout << "Default case" << std::endl;
	}
}
// [[nodiscard]] 属性

[[nodiscard]] int computeSomethingImportant() {
	return 42;
}
// [[maybe_unused]] 属性

void someFunction([[maybe_unused]] int unusedParam) {
	[[maybe_unused]] int unusedVar = 10;
	// 当你确实有意定义⼀个可能不会使⽤的变量（例如，

		// 条件编译、为了清晰的代码结构、或者为了将来扩展）时，避免编译器产⽣烦⼈的警告

		// 不会产⽣未使⽤变量的警告

}
int main() {
	checkValue(1);
	// 忽略[[nodiscard]]结果的警告

	computeSomethingImportant();  // 编译器可能会警告

	// 正确的⽅式是使⽤返回值

	int result = computeSomethingImportant();
	std::cout << "Result: " << result << std::endl;
	someFunction(100);
	checkValue(1);
	return 0;
}

 

💨 [[likely]] 和 [[unlikely]]

  • [[likely]]和[[unlikely]] (C++20)，这两个属性是用于向编译器提供分支预测的提示，帮助编译器有害代码。使其在条件分支更可能发生或者是更不可能发生时，生成小路更高的机器代码。
  • CPU采用的是流水线技术，可以余弦获取解码后续的指令。遇到条件分支（if /switch）时，CPU必须猜测会走那条路径一边提前加载指令。如果猜错了。就需要清空流水线，重新加载指令，这会带来显著的性能惩罚（称为分支预测错误惩罚！）
  • [[likely]]:向编译器指示，它所在代码上的路径（例如：if语句的true分支， 或switch语句的某个case）。准规定这些属性只是一个提示，编译器可以完忽略掉他们。编译器可能根据自己的分析结果来覆盖自己的提示。
  • 不要滥用！只有在你有充分的理由证明某一个分支确定存在极端概率的偏差时，才能使用这些属性。盲目的使用可能会误导编译器，适得其反，反而导致性能下降。
import std;

int process(int value) {
	if (value > 0) [[likely]] {
			//这个分⽀在绝⼤多数情况下都会执⾏

			return value * 2;
	}
	else [[unlikely]] {
		// 错误处理路径，很少执⾏
			//log_error("Invalid value");
			return -1;
	}
	
}
void func()
{
	int status = 0;
	std::cin >> status;
	// 也可以⽤于switch语句

			switch (status) {
			case 1: [[likely]]
				//handle_success();
				break;
			case 2: [[unlikely]]
				//handle_error();
				break;
			default: [[unlikely]]
				// handle_unknown();
				break;
			}
}
int main()
{
	process(1);
	func();
	return 0;
}

 

🦩[[no_unique_address]]

用于优化类的内存布局。他告诉编译器：被修饰的非静态数据空乘员可能不需要在对象中拥有对立的地址空间，空成员就是非静态成员变量的的类对象。

  • 在当前C++ 中，任何的对象（即使是空类class Empty{};）都必须拥有一个地址空间。这意味着<strong>sizeof</strong>至少为1， 以确保在数组中，每一个对象都能被区分开。当你有一个类包含多个空类类型的成员时，每个空成员都会至少占用一个字节，再加上内存对齐，会导致大量内存浪费。
  • 空基类优化（EBO-Empty Base Optimization）, C++ 标准允许编译器对空基类进行优化。如果一个类继承自空基类，编译器可以不为基类子对象分配任何空间，使其大小为0
  • [[no_unique_address]] 的作用类似EBO 的优化，它可以应用于非静态数据成员，提示编译器：如果该成员为空，则不需要为其分配一个唯一的地址。他允许这个成员与类中的其他非静态成员共性地址空间。
  • 要注意如果是相同类型的多个空乘员，这个优化可能会失效。
import std;
struct Empty1 {};
struct Empty2 {};
// 4 bytes
struct Foo {
	int x;
	[[no_unique_address]] Empty1 e1;  // 可能被优化掉
	[[no_unique_address]] Empty2 e2;  // 可能被优化掉

};
// 空基类优化(EBO - Empty Base Optimization)
struct Fxx : Empty1 {
	int x;
};
struct Fyy : Empty1, Empty2 {
	int y;
};
// 实践使⽤场景

template<typename Allocator = std::allocator<int>>
class Vector {
private:
	int* data;
	size_t size;
	size_t capacity;
	[[no_unique_address]] Allocator alloc; // allocator没有⾮静态成员变量
};

int main()
{
	std::cout << sizeof(Empty1) << std::endl;
	std::cout << sizeof(Foo) << std::endl;
	// EBO可以优化⼀个空基类，多个空基类就不⾏了
	std::cout << sizeof(Fxx) << std::endl;
	std::cout << sizeof(Fyy) << std::endl;
	std::cout << sizeof(std::allocator<int>) << std::endl;
	return 0;
}

 

这个编译优化的实现是否有效取决于具体的编译器，这个属性优化在一些主流的编译器上甚至都还没有实现。
